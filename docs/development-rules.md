
# Development Rules

Before making changes:

1. Read claude.md
2. Read game-design.md
3. Read roadmap.md

## Coding Principles

* Keep code simple.
* Avoid unnecessary abstractions.
* Prefer readability over cleverness.
* Favor modular architecture.
* Avoid duplication.

## Project Constraints

* No backend.
* No database.
* Local Storage only.
* Offline-first.

## File Modification Rules

* Modify only files related to the requested task.
* Explain major architectural decisions.
* Avoid breaking existing functionality.

## Documentation Rules

Update documentation only when:

* Architecture changes
* New systems are introduced
* Game mechanics change

Do not update documentation for every small implementation detail.
